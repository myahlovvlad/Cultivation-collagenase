## 1. Архитектура пользовательского взаимодействия — пять операционных потоков

Система BioCult-KB v2 реализует **пять процессных потоков**, каждый из которых решает конкретную задачу пользователя. Потоки независимы на уровне UI, но связаны через единую базу данных и AuditTrail.

```
┌─────────────────────────────────────────────────────────────────────┐
│                     BioCult-KB v2 — UI Layer                        │
├──────────┬──────────┬──────────┬──────────┬───────────┬────────────┤
│ Поток 1  │ Поток 2  │ Поток 3  │ Поток 4  │ Поток 5   │ Поток 6    │
│ Настройка│ Мониторинг│ Прогноз │ Анализ   │ Аудит /   │ OMICS /    │
│ процесса │ (live)   │ процесса │ сбоя     │ Валидация │ Геном      │
└──────────┴──────────┴──────────┴──────────┴───────────┴────────────┘
```

Каждый поток соответствует набору FastAPI эндпоинтов в `web_app/main.py` и набору экранов в SPA-интерфейсе.

## 2. Поток 1 — Настройка процесса («Что я планирую культивировать?»)

### 2.1 Операционное описание

Поток выполняется **до запуска биореактора**. Цель — сформировать полное техническое задание на процесс: штамм, среда, режим, граничные условия, предсказание выхода. Все решения этого этапа фиксируются в AuditTrail как `action_type = "process_design"`.

### 2.2 Шаги оператора

**Шаг 1.1 — Регистрация штамма**
- UI: раздел «Штаммы» → форма создания
- API: `GET /api/strains`
- Поля: `species`, `collection_number` (ВКПМ F-347), `biosafety_level`, `reactivation_rule`
- Результат: Запись в таблице `Strain`; штамм доступен при создании BatchRun

**Шаг 1.2 — Определение состава среды**
- UI: раздел «Среды» → «Новая среда» или выбор из справочника
- API: `GET /api/media`
- Компоненты задаются через форму компонентов: меласса (г/л), пептон (г/л), коллагеновый субстрат (г/л), минеральный фактор, корректоры pH
- Базовый справочник содержит: `MOLASSES20_PEPTONE85_V1` (референсная), `Оптимизированная меласса-пептон` (reference status)
- До инокуляции система моделирует среду как **буферный раствор** — расчёт буферной ёмкости выполняется модулем `ph_corrector.py`

**Шаг 1.3 — Конфигурация биореактора**
- UI: раздел «Биореактор» → «Геометрия и уставки»
- API: `GET /api/bioreactor-scene`
- Параметры геометрии: D_vessel=130мм, H=180мм, V_working=1.7л, D_imp=40мм, 3 барботажных дефлектора
- Уставки: Температура 25°C, pH 7.1–7.5, rpm 220, vvm 0.8, среда MOLASSES20_PEPTONE85_V1
- Результат: `BioreactorScene` с `CultivationSetpoints` — используется как начальное состояние для всех расчётных модулей

**Шаг 1.4 — Оптимизация среды (in silico)**
- UI: раздел «Оптимизатор среды» → ввод параметров → «Рассчитать»
- API: `POST /api/media-optimization` (вход: `MediaOptimizationInput`)
- Модуль `cell_process.py` выполняет многофакторный скоринг по 6 клеточным программам и 9 переменным среды
- Результат: `ProcessProjection` — предсказание биомассы, активности коллагеназы, pO₂, вязкости, времени сбора
- Граиндиозный перебор: `optimize_cultivation_strategy()` проверяет 4860 комбинаций параметров — занимает ~2–3 секунды

**Шаг 1.5 — Выбор оптимального сценария**
- API: `GET /api/cultivation-optimizer`
- Система возвращает топ-10 сценариев, ранжированных по целевой функции
- Текущий оптимум: меласса 18 г/л, пептон 95 г/л, коллаген 8 г/л, аэрация 0.9 vvm, 280 rpm → прогноз 107.2 U/mL
- **После v2:** Сценарий также проходит через транскриптомный FBA (`/api/transcriptome/fba`) — геномно-взвешенный прогноз

## 3. Поток 2 — Мониторинг в реальном времени («Что происходит прямо сейчас?»)

### 3.1 Операционное описание

Поток активен **во время культивирования**. Стенд датчиков обновляется циклически (рекомендуемый интервал: 30–60 с для inline-датчиков, 12–24 ч для offline-измерений). Ключевые модули: `calculations.py` (вычисление KLa, метрик), `recommendations.py` (правила экспертной системы).

### 3.2 Структура стенда датчиков (t=0ч, начальное состояние)

| Параметр | Значение t=0 | Тип | Норма / Риск |
|----------|-------------|-----|--------------|
| Температура | 24.8 °C | Inline PT-100 | Норма: 24–26°C |
| pH | 7.35 | Inline электрод | Норма: 7.1–7.5 |
| pO₂ | 92% | Inline полярограф | **Риск: >85% = избыток O₂** |
| pCO₂ | 0.3% | Inline ИК-газоанализатор | Норма: <3% |
| Перемешивание | 180 rpm | Привод с обратной связью | Контроль |
| Уровень | 71.1% | Ультразвуковой | Норма: 68–74% |
| Плотность | 1.018 г/мл | Вибрационный денсиметр | Норма |
| Вязкость | 1.18 мПа·с | Ротационный вискозиметр | Норма: <2.5 |
| Давление | 1.02 bar | Манометр | Норма: 1.0–1.1 |
| Влажность | 58% | Гигрометр (газ выхода) | Норма |
| Электропроводность | 8.4 mS/cm | Кондуктометр | Норма |
| Биомасса (inline) | 0.12 г/л | Рассеяние ИК (опция) | Контроль |

Профиль датчиков по времени (7 точек, 0–144ч) уже закодирован в `BioreactorScene.profile` как эталонный референс.

### 3.3 Расчёт KLa (workflow)

**Метод динамического отклика pO₂ (offline, после снятия аэрации):**

1. Оператор останавливает аэрацию → pO₂ начинает падать
2. Через 2–3 мин возобновляет аэрацию → pO₂ восстанавливается
3. Логирует серию значений pO₂(t) в форму `/api/calculate`
4. Система вычисляет KLa по уравнению: \[ k_La = \frac{dC_{O_2}/dt}{C_{O_2}^* - C_{O_2}} \]
5. Результат сохраняется в профиль батча

**После v2 — физический KLa из van't Riet:** `dfba_engine.compute_kLa(rpm, vvm, viscosity_mpa_s)` обновляет расчёт при каждом изменении rpm или вязкости, не требуя offline-измерения.

### 3.4 Логика алертов экспертной системы

Текущие правила в `recommendations.py` генерируют рекомендации при:
- pO₂ < 25% → «Проверить расход воздуха, rpm, пену, вязкость и морфологию»
- Время сбора ≈ 144ч → «Рассматривать как основную точку»
- Меласса = 20 г/л → «Провести A/B-тест 20 vs 30 г/л»

**После v2 — расширенные правила (физически обоснованные):**
```
pO₂ < 25% И kLa < 30 h⁻¹ И вязкость > 2.5 мПа·с →
  АЛЕРТ: «Кислородное лимитирование: вязкостная блокировка массопереноса.
          Немедленно: повысить rpm до 260+, проверить морфологию мицелия.
          Не повышать субстратную нагрузку до восстановления pO₂ > 35%.
          Расчётный резерв OTR при 260 rpm: +18%. Уверенность: 74%»
          AuditTrail: action_type="alert", evidence={kLa, pO2, viscosity, mu_hat}
```

## 4. Поток 3 — Предиктивный анализ («Что будет через N часов?»)

### 4.1 Операционное описание

**Текущее состояние v1:** `process_simulation.py` выполняет Euler-цикл с эмпирическим μ = `0.006 + 0.060 * capacity_index * ...` — коэффициенты подобраны вручную, без физического обоснования.

**Целевое состояние v2:** `dfba_engine.run_dfba_step()` вычисляет μ через Monod-кинетику + COBRApy FBA на каждом шаге. Профиль содержит новые поля: `dfba_mu_h`, `kLa_h`, `OTR_mmol_l_h`.

### 4.2 Workflow запуска симуляции

**Шаг 3.1 — Выбор режима**
- API: `GET /api/process-modes` → `batch | fed_batch | continuous`
- Batch (144ч): V = const = 1.7л, нет добавок субстрата
- Fed-batch: подпитка по расписанию (`feed_rate_l_h`, `feed_start_h`)
- Continuous: подача + слив, `bleed_rate_l_h`

**Шаг 3.2 — Параметры симуляции**
- API: `POST /api/process-simulation`
- Входные данные (`ProcessSimulationInput`): `process_mode`, `medium`, `duration_h=144`, `step_h=12`, `initial_biomass_g_l=0.12`
- Шаг интегрирования: 12ч (рекомендован для batch); для fed-batch — 4–6ч вблизи точек подпитки

**Шаг 3.3 — Интерпретация профиля**

Каждая точка профиля содержит:

| Поле | Единица | Источник v1 | Источник v2 |
|------|---------|-------------|-------------|
| `biomass_g_l` | г/л | empirical μ | dFBA μ |
| `product_u_ml` | U/mL | empirical qP | FBA qP |
| `pO2_percent` | % | линейная аппроксимация | kLa + OTR уравнение |
| `viscosity_mpa_s` | мПа·с | `1.05 + 0.34*X + ...` | power-law реология |
| `fba_mu_capacity_h` | ч⁻¹ | FBA objective | dFBA μ (замена) |
| `kLa_h` | ч⁻¹ | отсутствует | van't Riet |
| `OTR_mmol_l_h` | ммоль/л/ч | отсутствует | kLa * (C* - DO) |

**Шаг 3.4 — Транскриптомно-взвешенный прогноз (v2)**
- API: `POST /api/transcriptome/fba`
- Применяет E-Flux к GEM: TPM-веса ограничивают верхние границы реакций
- Прогноз становится геномно-обоснованным: «Ген `prtA` (S8 протеаза) активен при текущем pO₂ и пептоне. Прогноз qP +12% vs unconstrained FBA»
- Метаданные ответа: `gene_coverage`, `n_reactions_constrained`, `data_source`

### 4.3 Критические контрольные точки процесса

| Время | Фаза | Контрольный параметр | Действие при отклонении |
|-------|------|---------------------|------------------------|
| 0–12ч | Адаптация инокулюма | pO₂ > 70%, pH стабильный | Если pO₂ < 50% — загрязнение или ошибка инокуляции |
| 12–48ч | Экспоненциальный рост | pO₂ > 35%, X растёт | rpm+20 при pO₂ < 30% |
| 48–72ч | Продуктивная фаза | вязкость < 2.5, kLa > 50 | Снизить нагрузку субстрата |
| 72–120ч | Накопление продукта | pH 7.0–7.4, pCO₂ < 5% | Корректор pH (NaOH 2M) |
| 120–144ч | Окно сбора | активность фермента максимум | Отбор 10мл каждые 12ч |

## 5. Поток 4 — Анализ отклонений («Почему процесс вышел за границы?»)

### 5.1 Операционное описание

Поток активируется **при любом отклонении** от нормы (алерт от экспертной системы). Цель — root cause analysis с геномным + процессным обоснованием.

### 5.2 Методология анализа (три уровня)

**Уровень 1 — Немедленная диагностика (< 5 мин)**

Оператор вводит текущие показания в `/api/dfba/step` (v2) или `/api/calculate` (v1):
```
Вход: biomass=3.1, molasses=7.0, nitrogen=5.8, DO=0.30, rpm=260, vvm=0.8, viscosity=2.35
Выход: mu=0.008 h⁻¹, kLa=52.3 h⁻¹, OTR=3.2 ммоль/л/ч, status=optimal
```
Если `status = "degraded"` → GEM infeasible → субстрат исчерпан или pH вне диапазона.

**Уровень 2 — Ретроспективный анализ профиля (< 30 мин)**

Сравнение текущего профиля с эталонным `BioreactorScene.profile`:

| Параметр | Эталон t=72ч | Факт t=72ч | Отклонение | Вероятная причина |
|----------|-------------|-----------|-----------|------------------|
| pO₂ | 30% | 18% | -40% | ↑ вязкость → ↓ kLa |
| Биомасса | 3.1 г/л | 3.8 г/л | +23% | Избыток N-субстрата |
| Вязкость | 2.35 мПа·с | 3.4 мПа·с | +45% | Мицелиальная агрегация |
| pH | 7.08 | 6.85 | -3.3% | Недостаточная буферность |

**Уровень 3 — Геномно-процессная причинно-следственная цепочка**

`POST /api/system-biology-model/evaluate` → `SystemBiologyConditionResult`:
```
Входные условия (фактические) →
  CellularPrograms: вязкость 71%, стресс 45% (watch) →
  Прогноз: переключение от роста к стресс-ответу и вторичным метаболитам →
  Геномный слой: BGC-гены (laeA, veA) получают более высокий вес при стрессе →
  Рекомендация: снизить пептон на 15%, повысить vvm до 1.1
```

## 6. Поток 5 — Аудит и валидация (21 CFR Part 11 / GAMP5)

### 6.1 Операционное описание

Каждое решение системы и каждое действие оператора должно иметь **неизменяемую запись** с SHA-256 хэшем. Это обеспечивает соответствие 21 CFR Part 11 и EU GMP Annex 11.

### 6.2 Структура AuditRecord (v2, новая таблица в `models.py`)

```
AuditRecord:
  id            — уникальный номер
  timestamp     — UTC-время события
  user          — имя оператора / "system"
  action_type   — fba_step | recommendation | param_change | alert | simulation_complete
  module        — dfba_engine | transcriptome | gem_validation | expert_system
  recommendation — текст рекомендации (рус.)
  evidence_json — {kLa, pO2, mu, biomass, model_version, confidence}
  confidence    — 0.0–1.0
  decision      — accepted | rejected | pending
  record_hash   — SHA-256(timestamp + user + action + evidence)
  outcome_json  — заполняется после следующего измерения (замыкание петли)
```

### 6.3 Workflow верификации записи

1. `GET /api/audit/records?limit=50` → список записей
2. `GET /api/audit/verify/{record_id}` → пересчёт SHA-256 → сравнение со stored_hash
3. Если `"valid": false` → запись была изменена → нарушение целостности → инцидент

### 6.4 GAMP5 IQ/OQ/PQ — автоматический запуск

```bash
# IQ: проверка установки
pytest tests/iq_oq_pq/test_IQ.py -v -m iq

# OQ: проверка операционной корректности
pytest tests/iq_oq_pq/test_OQ.py -v -m oq

# PQ: проверка производительности
pytest tests/iq_oq_pq/test_PQ.py -v -m pq
```

**PQ-критерии приёмки (проектное пространство A. sydowii ВКПМ F-347):**
- PQ-01: Финальная биомасса 2.0–4.5 г/л
- PQ-02: Коллагеназная активность 60–150 U/mL
- PQ-03: pO₂ ≥ 25% при t=72ч
- PQ-04: kLa = 15–50 ч⁻¹ при 180 rpm / 0.8 vvm
- PQ-05: Богатая среда даёт ≥1.5× продукт vs бедная
- PQ-06: dFBA не входит в degraded mode при нормальном батче

## 7. Поток 6 — OMICS-пайплайн («Что написано в геноме?»)

### 7.1 Операционное описание

Поток предоставляет геномное обоснование для всех прогнозов системы. Текущий статус: аннотационный скаффолдинг через `omics_pipeline.py`, без загруженных данных. После загрузки GCA_001890705.1 геном переходит в «активный» статус.

### 7.2 Последовательность работы с OMICS

**Шаг 6.1 — Реестр источников**
- API: `GET /api/omics/registry`
- Статусы: 5 сборок генома (5 проверено), 1 транскриптом (PRJNA542911), 5 метаболомических источников (2 проверено, 3 кандидата)
- Инструменты: `datasets`, `esearch`, `efetch`, `prefetch`, `fasterq-dump` — проверяются на установку

**Шаг 6.2 — Манифест проекта**
- API: `POST /api/omics/manifest`
- Вход: `OmicsProjectRequest` (genome_accession, transcriptome_bioproject)
- Выход: manifest.json, assets_index.json, команды для скачивания

**Шаг 6.3 — Скачивание генома**

Команда из манифеста (для выполнения на машине с NCBI Datasets):
```bash
datasets download genome accession GCA_001890705.1 \
  --include genome,gff3,rna,cds,protein,gbff,seq-report \
  --filename "data/omics/aspergillus_sydowii/raw/GCA_001890705.1_ncbi_dataset.zip"
```

**Шаг 6.4 — Транскриптом (PRJNA542911)**
```bash
esearch -db sra -query "PRJNA542911[BioProject]" | \
  efetch -format runinfo > reports/PRJNA542911_runinfo.csv
```
- API: `POST /api/omics/discover-transcriptome`
- После получения RunInfo — загрузка reads через скрипт `PRJNA542911_download_reads.sh`

**Шаг 6.5 — Загрузка TPM в E-Flux (v2)**
- API: `POST /api/transcriptome/upload-tpm` (multipart CSV)
- Формат: `gene_id,TPM` или матрица (gene × samples)
- После загрузки: вызов `/api/transcriptome/fba` использует реальные TPM вместо proxy

**Шаг 6.6 — Отчёт биосинтеза**
- API: `POST /api/omics/biosynthesis-report`
- Возвращает аннотационные доказательства по 6 категориям: протеазы, секреция, транспорт, ЦУМ, вторичный метаболизм, клеточный цикл
- Каждая категория: `score`, `evidence_count`, `sources_count`, `confidence`

## 8. Инструктивно-методологический workflow проведения измерений

### 8.1 Подготовительный этап (за 24–48ч до запуска)

```
[ ] 8.1.1  Проверить стерилизацию биореактора (автоклав 121°C, 20 мин)
[ ] 8.1.2  Приготовить питательную среду MOLASSES20_PEPTONE85_V1:
           - Меласса 20.0 г/л (сахар-сырец, ρ ≈ 1.32 г/мл)
           - Пептон 85.0 г/л (гидролизат казеина, N ≈ 14%)
           - Коллагеновый субстрат 5.0 г/л (коллаген I типа, нативный)
           - Минеральный раствор (1× MnSO₄, FeSO₄, ZnSO₄, MgSO₄)
           - Корректировка pH до 7.2 ± 0.1 (2M NaOH / 2M H₂SO₄)
[ ] 8.1.3  Загрузить 1.6 л среды в биореактор (10% объёма — под инокулят)
[ ] 8.1.4  Откалибровать датчики:
           - pH-электрод: буферные стандарты pH 4.0 и 7.0
           - pO₂-электрод: 0% (N₂) и 100% (воздух насыщенный)
           - Денсиметр: деионизированная вода (1.000 г/мл)
           - Вискозиметр: глицерин-эталон (соответствующий диапазон)
[ ] 8.1.5  Внести начальные данные в систему:
           API: POST /api/strains (если штамм не зарегистрирован)
           API: POST /api/media-optimization (для предварительного прогноза)
           API: POST /api/process-simulation (для базового профиля t=0..144)
```

### 8.2 Запуск процесса (t = 0ч)


### 8.4 Измерение коллагеназной активности (основной целевой показатель)

**Метод: флуориметрический (MCA-Pro-Leu-Gly-Pro-D-Arg-Dpa-Ala-Arg-NH₂)**

Шаг 1: Подготовить реакционную смесь:
```
- Буфер: 50 мМ Tris-HCl, pH 7.5, 5 мМ CaCl₂, 0.15M NaCl
- Субстрат: 10 мкМ MCA-флуоресцентный пептид
- Объём: 200 мкл / лунку (96-луночный планшет)
```

Шаг 2: Добавить пробу (10 мкл культуральной жидкости, центрифугированной 5000 rpm).

Шаг 3: Инкубация 30 мин, 37°C, в темноте.

Шаг 4: Измерение флуоресценции: λex=325нм, λem=393нм (спектрофлуорометр).

Шаг 5: Расчёт активности:
\[ A \, [\text{U/mL}] = \frac{(F_{sample} - F_{blank}) \cdot V_{total}}{F_{std} \cdot V_{sample} \cdot t} \]
где 1 U = гидролиз 1 нмоль субстрата/мин при 37°C.

Шаг 6: Внести результат в систему → `POST /api/calculate` с полем `protein` и `sugar` → пересчёт удельной активности.

### 8.5 Расчёт KLa (динамический метод — протокол)

**Условие выполнения:** рост биомассы ≥ 0.5 г/л (значимый OUR), pO₂ = 40–60%

```
1. Записать baseline pO₂ (при стационарном перемешивании + аэрации)
2. Отключить аэрацию (сохранить rpm)
3. Записывать pO₂ каждые 30 с в течение 5–8 мин
   → pO₂ падает: линейный участок → OUR = d(DO)/dt
4. Включить аэрацию (тот же rpm)
5. Записывать pO₂ каждые 30 с до восстановления baseline
   → kLa = OUR / (DO* - DO_steady_state)
6. Внести серию в API: POST /api/calculate
   {observations: [{time_h: ..., pO2: ...}, ...]}
   Система рассчитывает KLa и логирует в AuditTrail
```

Граничные условия корректности: \( R^2 \geq 0.95 \) линейной аппроксимации на спаде; если меньше — повторить измерение (мешает пенообразование или протечка).

### 8.6 Действия при ключевых отклонениях

**Сценарий A: pO₂ снижается ниже 25% при t < 60ч**
```
1. API: POST /api/dfba/step (внести текущие значения)
   → Проверить: status=degraded? (кислородное лимитирование)
2. Немедленно: rpm +20 (максимум 280 rpm без разрушения мицелия)
3. Проверить барботажную трубку (засорение пеной или агрегатом)
4. Если вязкость > 2.5 мПа·с: НЕ повышать субстратную нагрузку
5. Зафиксировать в AuditTrail:
   POST /api/audit/log
   {action_type: "param_change", module: "bioreactor",
    recommendation: "rpm+20 при pO2<25%",
    evidence: {pO2: 23.5, viscosity: 2.7, kLa: 31, time_h: 52},
    confidence: 0.82, decision: "accepted"}
```

**Сценарий B: pH < 7.0 при t > 48ч**
```
1. Активировать автоматический pH-регулятор (если не активен)
2. Разовая коррекция: 2M NaOH до pH 7.1–7.2
3. Проверить: достаточность буфера среды
   (признак исчерпания: требуется > 5 мл NaOH/12ч)
4. Снизить скорость аэрации на 15% (↓ CO₂ удаление)
5. Зафиксировать в AuditTrail: action_type="ph_correction"
```

**Сценарий C: Биомасса не растёт при t = 12–24ч (Lag > 24ч)**
```
1. Проверить жизнеспособность инокулята (микроскопия → прорастание спор)
2. Если < 50% прорастания → перезапустить (новый инокулят)
3. Если > 50% → проверить pH (токсичность при pH < 6.5)
4. Если контаминация (микроскопия) → остановить процесс
5. Зафиксировать: action_type="deviation", decision="restart"
```

## 9. Методология работы с расчётной формой (`POST /api/calculate`)

### 9.1 Входная структура (CalculationInput)

Форма принимает серию объектов `ObservationBase`:
```json
{
  "observations": [
    {
      "time_h": 48.0,
      "pO2_percent": 38.0,
      "biomass_g_l": 1.9,
      "kla_h": null,
      "protein_mg_ml": 0.82,
      "sugar_g_l": 11.0
    }
  ]
}
```

### 9.2 Что рассчитывает система

Модуль `calculations.py` вычисляет:
- **KLa** по серии pO₂ значений (если переданы)
- **Удельная скорость роста** μ = ln(X₂/X₁) / (t₂ - t₁) — из двух последовательных биомасс
- **Удельная производительность** qP = ΔP / (X · Δt) — из активности и биомассы
- **Выход продукта** Yp/s = ΔP / ΔS — из активности и субстрата
- **Степень использования субстрата** (S₀ - S) / S₀ × 100%

### 9.3 Связь с экспертной системой

После расчёта результаты автоматически передаются в `recommendations.py`, которая генерирует контекстные рекомендации на русском языке. В v2 рекомендации дополняются:
- Ссылкой на ген (если E-Flux показывает узкое место)
- Физически обоснованным механизмом (через dFBA)
- Уровнем уверенности модели (0.0–1.0)
- Временем реакции (немедленно / следующее измерение / плановое)

## 10. Критерии сбора культуральной жидкости (окно процесса)

### 10.1 Основные критерии (все должны выполняться)

| Критерий | Целевое значение | Метод определения |
|----------|-----------------|-------------------|
| Активность коллагеназы | максимум или ≥90 U/mL | Флуориметрический метод |
| dA/dt | ≤ 0 (плато или снижение) | Два последовательных измерения |
| Биомасса | 3.0–4.0 г/л | OD₆₀₀ или inline |
| pO₂ | > 25% | Inline полярограф |
| Вязкость | < 2.8 мПа·с | Вискозиметр |
| pH | 7.0–7.4 | Inline электрод |

### 10.2 Расчётный прогноз времени сбора (API)

```
POST /api/media-optimization
→ projection.recommended_harvest_time_h: 120 (если enzyme_index ≥ 78 AND viscosity_risk < 65)
                                         144 (иначе)
```

### 10.3 Процедура сбора

```
t = 120–144ч (окно):
1. Отключить мешалку → дать биомассе осесть 15 мин (если мицелиальная)
2. Отобрать 30 мл культуральной жидкости через пробоотборник
3. Центрифугировать: 8000 rpm, 4°C, 15 мин
4. Супернатант — целевой продукт (внеклеточная коллагеназа)
5. Измерить: активность, белок (Bradford), pH, вязкость
6. Зафиксировать финальные показания в системе:
   POST /api/calculate + POST /api/audit/log (action_type="harvest")
7. Хранить при -20°C (краткосрочно) или лиофилизировать
```
