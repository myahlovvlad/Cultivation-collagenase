# Руководство по эксплуатации BioCult-KB Web Prototype

Версия документа: 1.0  
Дата: 2026-05-14  
Система: BioCult-KB / *Aspergillus sydowii*  
Назначение: эксплуатация веб-прототипа для поддержки культивирования продуцента коллагенолитической активности.

## 1. Назначение системы

BioCult-KB Web Prototype предназначен для инженерной и экспертной поддержки культивирования *Aspergillus sydowii* в лабораторном биореакторе.

Система позволяет:

- просматривать 3D-сцену биореактора и модельный профиль процесса;
- отслеживать показания датчиков и временные ряды;
- рассчитывать метрики по контрольным точкам;
- получать рекомендации экспертной системы;
- оценивать влияние состава среды на биомассу, кислородный режим, вязкость и коллагенолитический потенциал;
- использовать GEM-lite COBRA/SBML модель для FBA-оценки ограничений среды;
- запускать процессную симуляцию для периодического, отъёмно-доливного и непрерывного режимов;
- вести подготовительный OMICS-контур источников и артефактов.

## 2. Границы применимости

Текущая версия является инженерным прототипом v1.

Ограничения:

- GEM-lite не является полноразмерной курируемой GEM-моделью;
- FBA использует прокси-метаболиты и прокси-стехиометрию;
- коллагенолитическая активность выводится из доказательств протеаз/пептидаз и секреции, если нет прямой аннотации collagenase;
- клеточный цикл представлен как evidence-layer, без ODE/event-модели;
- физико-химическая динамика среды рассчитывается приближённо;
- результаты должны валидироваться экспериментальными наблюдениями.

## 3. Требования к рабочей среде

Рекомендуемая среда:

- Windows 10/11;

- доступ к локальным файлам проекта;
- браузер Chrome, Edge или Firefox.

Python-зависимости указаны в:

Ключевые зависимости:

## 4. Запуск приложения

Из корня репозитория выполните:

```powershell
cd BioCult-KB_Aspergillus_sydowii
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r web_app\requirements.txt
$env:PYTHONPATH = (Resolve-Path .).Path
python -m uvicorn web_app.main:app --host 127.0.0.1 --port 8000
```

Откройте в браузере:

```text
http://127.0.0.1:8000/
```

Если браузер показывает `ERR_CONNECTION_REFUSED`, значит сервер не запущен или порт занят другим процессом.

## 5. Основные разделы интерфейса

### 5.1 3D-визуализация

Раздел показывает стеклянный лабораторный биореактор, мешалку, барботаж, уровень среды и модельное состояние процесса.

Основные элементы:

- время процесса;
- биомасса;
- pO2;
- KLA;
- слайдер хода процесса;
- кнопки `Пуск` и `Сброс`.

Слайдер `Ход процесса` меняет текущую модельную точку процесса. Кнопка `Пуск` запускает автоматическое проигрывание профиля.

### 5.2 Стенд датчиков

Стенд отображает модельные показания:

- перемешивание;
- pH;
- pO2;
- pCO2;
- уровень;
- плотность;
- вязкость;
- температура;
- давление;
- влажность;
- электропроводность;
- биомасса.

Индикаторы состояния:

- `Норма` - показатель в рабочем диапазоне;
- `Контроль` - показатель требует внимания;
- `Риск` - показатель вышел в критический диапазон.

### 5.3 Временные ряды

Раздел визуализирует динамику групп показателей:

- газы;
- культура;
- контур управления;
- среда.

График используется для быстрой оценки тренда, а не как метрологически заверенный отчёт.

### 5.4 Экспертная система

Экспертная система показывает:

- фазу процесса;
- уровень уверенности;
- интегральный риск;
- события;
- действия оператора.

Рекомендации следует сверять с реальными наблюдениями, журналом оператора и лабораторными измерениями.

### 5.5 Геном → процесс

Слайдеры задают проектные уставки среды и режима:

- меласса;
- пептон;
- коллагеновый субстрат;
- минеральный фактор;
- pH;
- аэрация;
- перемешивание.

Эти значения являются входом для прогнозных расчётов и не обязаны совпадать с текущими модельными датчиками.

### 5.6 Системно-биологический слой

Раздел показывает:

- сводку генома;
- категории аннотационных доказательств;
- оценку условий;
- предупреждения и интерпретации.

Особое правило интерпретации:

Если прямой keyword-hit `collagenase` в GBFF/protein отсутствует, система не утверждает, что коллагеназа найдена. Коллагенолитический потенциал выводится только косвенно: через протеазы/пептидазы, секрецию и условия культивирования.

### 5.7 COBRA / SBML

Раздел отображает:

- статус SBML-модели;
- количество реакций, метаболитов и генов;
- результат FBA;
- лимитирующие обменные реакции;
- профиль процессной симуляции;
- уравнения материального баланса.

Доступные режимы:

- периодический;
- отъёмно-доливной;
- непрерывный.

Ссылка `Скачать SBML` выгружает файл GEM-lite модели.

### 5.8 OMICS-пайплайн

Раздел предназначен для подготовки источников:

- доступ генома;
- BioProject транскриптома;
- источник метаболома;
- реестр источников;
- статус локальных инструментов;
- выходные файлы.

Если инструменты NCBI отсутствуют, интерфейс показывает `нет`. Это не блокирует основные расчёты, но ограничивает автоматическую загрузку новых OMICS-данных.

## 6. API для проверки

Основные endpoint:

```text
GET  /api/gem/summary
POST /api/gem/fba
GET  /api/gem/sbml
POST /api/process-simulation
GET  /api/system-biology-model
POST /api/system-biology-model/evaluate
GET  /api/bioreactor-scene
```

Пример FBA-запроса:

```json
{
  "medium": {
    "molasses_g_l": 20,
    "peptone_g_l": 85,
    "collagen_g_l": 5,
    "mineral_factor": 1,
    "pH": 7.2,
    "aeration_vvm": 0.8,
    "rpm": 220
  },
  "objective": "balanced",
  "growth_floor": 0.05
}
```

## 8. Типовые ошибки и действия

### Ошибка: ERR_CONNECTION_REFUSED

Причина: сервер не запущен.

Действие:

```powershell
python -m uvicorn web_app.main:app --host 127.0.0.1 --port 8000
```

### Ошибка: 404 на `/api/gem/fba`

Причина: запущен старый серверный процесс.

Действие: остановить старый процесс и запустить сервер заново.

### Ошибка: COBRA/SBML degraded mode

Причина: отсутствуют или некорректно установлены COBRApy/libSBML/GLPK-зависимости.

Действие:

```powershell
pip install -r web_app\requirements.txt
```

### Ошибка: NCBI tools показывают `нет`

Причина: `datasets`, `esearch`, `efetch`, `prefetch`, `fasterq-dump` не установлены или не находятся в PATH.

Действие: установить NCBI Datasets, EDirect и SRA Toolkit, затем перезапустить приложение.

## 9. Правила эксплуатации и интерпретации

1. Не использовать v1 как единственный источник решения о реальном процессе.
2. Проверять прогнозы по журналу оператора и лабораторным измерениям.
3. Не интерпретировать косвенный протеазный сигнал как доказанную индивидуальную коллагеназу.
4. Для публикационных выводов требуется ручная курация GEM, валидация кандидатов и экспериментальные данные.
5. При изменении среды или режима фиксировать исходные параметры, время расчёта и версию модели.

## 10. Резервное копирование и данные

В git не включаются:

- большие NCBI zip-файлы;
- raw OMICS downloads;
- локальные SQLite базы;
- кэши тестов и runtime.

Перед переносом проекта на другую машину необходимо отдельно передать локальные NCBI-данные или повторно загрузить их через OMICS-пайплайн.
